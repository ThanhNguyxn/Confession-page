### 2. Configure Your Site

Edit both `index.html` and `confession.html` to replace all placeholders:

#### A. Basic Information

Find and replace these placeholders throughout **both files**:

| `[YOUR_CFS_WEBSITE_URL]` | "See More" button destination URL | "https://yoursite.com/confessions" |

**Quick Find & Replace in Text Editor:**

### 3. "See More" Button Configuration

The confession tracking page includes a "See More Confessions" button that directs users to your main confession platform.

**To configure:**
1. Open `confession.html`
2. Find line ~3158: `<a href="[YOUR_CFS_WEBSITE_URL]" target="_blank">`
3. Replace `[YOUR_CFS_WEBSITE_URL]` with your confession platform URL

**Examples:**
- External platform: `https://yourconfessionplatform.com`
- Same site page: `index.html#submit`
- Backend route: `/confessions/browse`

**To remove the button:**
- Delete lines 3155-3165 in `confession.html` (the entire "See More Button Section")

### 4. Form Submission Backend

The form currently submits to `#`. You need to:

1. Create a backend endpoint (PHP, Node.js, Python, etc.)
2. Update the form action in `index.html`:

```html
<!-- Find -->
<form action="#" method="POST">

<!-- Replace with -->

```

3. Process form data on your backend:
   - `category` - Category selected
   - `content` - Confession text
   - `photo` - Photo URL (optional)
   - `note` - Private note to admin

### 5. Database Setup

For tracking functionality, you'll need:

**Confessions Table:**
```sql
CREATE TABLE confessions (
    id INT AUTO_INCREMENT PRIMARY KEY,
    tracking_code VARCHAR(50) UNIQUE,
    category VARCHAR(50),
    content TEXT,
    photo_url VARCHAR(255),
    status ENUM('Pending', 'Posted') DEFAULT 'Pending',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## üöÄ Deployment

### Quick Start - Deploy to GitHub Pages

This is the easiest way to get your confession site online for free!

#### Step 1: Create GitHub Repository

1. Go to [GitHub](https://github.com) and sign in (or create an account)
2. Click the "+" icon in the top right ‚Üí "New repository"
3. **Repository Name**: Choose a name (e.g., `my-confession-page`)
4. **Visibility**: Choose "Public" (required for free GitHub Pages)
5. **DO NOT** initialize with README (we already have files)
6. Click "Create repository"

#### Step 2: Initialize Git and Push Your Code

Open Command Prompt/Terminal in your project folder (`D:\Code\cfs-page\`) and run:

```bash
# Initialize git repository
git init

# Add all files
git add .

# Commit your changes
git commit -m "Initial commit: Confession page setup"

# Link to your GitHub repository (replace with your repo URL)
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git

# Push to GitHub
git branch -M main
git push -u origin main
```

**Replace** `YOUR_USERNAME` and `YOUR_REPO_NAME` with your actual GitHub username and repository name.

#### Step 3: Enable GitHub Pages

1. Go to your repository on GitHub
2. Click "Settings" tab
3. Scroll down to "Pages" in the left sidebar
4. Under "Source", select **"Deploy from a branch"**
5. Under "Branch", select **"main"** and **"/ (root)"**
6. Click **"Save"**
7. Wait 1-2 minutes, then refresh the page
8. Your site URL will appear at the top: `https://YOUR_USERNAME.github.io/YOUR_REPO_NAME/`

#### Step 4: Update Your Site URL References

After deployment, update any `[YOUR_TRACKING_URL]` placeholders in `index.html` to point to your GitHub Pages URL.

### Deploy with Backend (Node.js + Express example):

**Note:** For form submissions to work, you need a backend. Here's a basic setup:

```bash
# Initialize Node.js project
npm init -y

# Install dependencies
npm install express body-parser cors

# Create server.js
```

**server.js example:**
```javascript
const express = require('express');
const bodyParser = require('body-parser');
const cors = require('cors');
const app = express();

app.use(cors());
app.use(bodyParser.json());
app.use(express.static('.')); // Serve static files

// Handle confession submission
app.post('/api/submit-confession', (req, res) => {
    const { category, content, photo, note } = req.body;
    
    // TODO: Save to database
    // For now, just return success
    const trackingCode = Math.random().toString(36).substring(2, 8).toUpperCase();
    
    res.json({
        success: true,
        trackingCode: trackingCode,
        message: 'Confession submitted successfully!'
    });
});

// Start server
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => {
    console.log(`Server running on http://localhost:${PORT}`);
});
```

**Run server:**
```bash
node server.js
```

### Continuous Deployment Setup

#### GitHub Actions (Auto-deploy on push):

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      
      - name: Deploy to GitHub Pages
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./
```

This will auto-deploy your site every time you push changes to GitHub!
